通过 GITHUB WEBHOOKS 实现自动部署

发表于 2019-01-15 | 分类于 工具与资源
前言

WEBHOOKS 其实就是当发生特定的事件时通知特定的服务。GITHUB WEBHOOKS 的特定事件指的是 GIT 事件，例如 PUSH。特定的服务指的就是用户自己配置的 POST 请求。

所以，要通过 GITHUB WEBHOOKS 来实现代码的自动部署，首先需要提供一个对外的 POST 接口，然后通过这个接口执行代码部署命令。
